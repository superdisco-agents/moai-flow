# MoAI-ADK 빠른 시작 가이드 (10분)

## 👋 환영합니다!

이 가이드는 **10분 안에** MoAI-ADK의 핵심 개념을 이해할 수 있도록 설계되었습니다.

**목표**: 에이전트, 스킬, 명령어, 스크립트, MCP 서버가 무엇이고 언제 사용하는지 빠르게 파악하기

## 🎯 핵심 개념 5가지

### 1. 에이전트 (Agents) - 전문 작업자

**한 문장 정의**: 특정 작업을 전문적으로 수행하는 실행 단위

**비유**: 회사의 팀원들
- expert-backend → 백엔드 개발자
- expert-frontend → 프론트엔드 개발자
- manager-tdd → TDD 프로세스 관리자

**언제 사용**:
- 도메인 전문 작업 (Backend, Frontend, Database 등)
- Multi-step 워크플로우
- 코드베이스 탐색 (Read-only)

**사용 예시**:
```python
# 백엔드 API 구현을 전문 에이전트에게 위임
Task(subagent_type="expert-backend",
     prompt="Implement REST API for user authentication")

# 코드베이스 탐색 (빠른 read-only)
Task(subagent_type="Explore",
     prompt="Find authentication handlers",
     thoroughness="medium")
```

**핵심 포인트**:
- ✅ 24개 에이전트 (5-Tier 계층)
- ✅ Task() 로 호출
- ✅ Resume 패턴으로 40-60% 토큰 절약 가능
- ❌ Read(), Write(), Edit() 직접 사용 금지 → 항상 Task() 위임

### 2. 스킬 (Skills) - 도메인 지식

**한 문장 정의**: 에이전트가 참고하는 Procedural Knowledge (절차적 지식)

**비유**: 회사의 업무 매뉴얼, 가이드북
- moai-lang-python → Python 개발 패턴
- moai-foundation-trust → TRUST 품질 기준
- moai-platform-baas → BaaS 통합 가이드

**언제 사용**:
- 도메인 패턴 참조 필요
- HOW-TO 가이드 제공
- 에이전트에게 지식 주입

**사용 예시**:
```yaml
# Agent YAML frontmatter에 스킬 선언
---
name: expert-backend
skills: moai-lang-python, moai-platform-baas, moai-foundation-trust
---

# 조건부 auto-load (Alfred가 자동 로드)
# 복잡도 >= Medium → moai-foundation-core 자동 로드
```

**핵심 포인트**:
- ✅ 128+ 스킬 (4-Tier 조직)
- ✅ Markdown 문서 형식
- ✅ Auto-load 트리거 (조건부 자동 로드)
- ❌ 실행 불가 (지식만 제공)

### 3. 명령어 (Commands) - 사용자 단축키

**한 문장 정의**: 복잡한 워크플로우를 한 번에 실행하는 사용자 명령어

**비유**: 회사의 표준 프로세스
- /moai:1-plan → 프로젝트 계획 수립
- /moai:2-run → TDD 구현 실행
- /moai:3-sync → 문서 동기화

**언제 사용**:
- 명시적 워크플로우 실행
- SPEC 생성 및 구현
- 문서화

**사용 예시**:
```bash
# 새 기능 SPEC 작성
/moai:1-plan "User authentication with JWT"

# TDD RED-GREEN-REFACTOR 사이클 실행
/moai:2-run SPEC-001

# 문서 동기화 및 PR 생성
/moai:3-sync SPEC-001
```

**핵심 포인트**:
- ✅ 7개 명령어 (/moai:0-3, 9, 99, cleanup)
- ✅ 4-Phase 실행 (Intent → Plan → Execute → Report)
- ✅ Config-driven (git_strategy.mode에 따라 동작 변경)
- ❌ 내부적으로도 Task()로 에이전트 위임 (직접 실행 X)

### 4. 스크립트 (Scripts) - 빠른 자동화

**한 문장 정의**: 이벤트 기반으로 자동 실행되는 one-shot 작업

**비유**: 회사의 자동화 규칙
- session_start → 출근 시 오늘 할 일 표시
- pre_commit → 커밋 전 자동 린트
- post_edit → 파일 수정 후 자동 포맷

**언제 사용**:
- 이벤트 기반 자동화
- 빠른 one-shot 작업
- Zero 토큰 비용 필요

**사용 예시**:
```python
# .claude/hooks/moai/session_start__show_project_info.py
def hook(ctx: HookContext) -> HookResponse:
    """세션 시작 시 프로젝트 정보 표시"""
    config = load_config()
    project_info = get_project_info()
    return HookResponse(
        success=True,
        output=format_project_info(project_info)
    )
```

**핵심 포인트**:
- ✅ Zero 토큰 비용
- ✅ 빠른 실행 (2초 timeout)
- ✅ Graceful degradation (실패해도 계속 진행)
- ❌ Stateless (상태 유지 불가)

### 5. MCP 서버 (MCP Servers) - 외부 통합

**한 문장 정의**: Stateful하게 외부 서비스를 통합하는 JSON-RPC 2.0 서버

**비유**: 회사의 외부 협력 파트너
- context7 → 실시간 기술 문서 제공업체
- playwright → 브라우저 자동화 대행사
- figma → 디자인 협업 플랫폼
- sequential-thinking → 복잡한 분석 컨설턴트

**언제 사용**:
- 상태 유지 필요 (Stateful)
- 외부 서비스 통합
- Real-time 알림
- Cross-session memory

**사용 예시**:
```typescript
// 1. 라이브러리 ID 조회
mcp__context7__resolve-library-id("react")

// 2. 최신 API 문서 가져오기
mcp__context7__get-library-docs("/facebook/react/v19.0.0")

// 3. 브라우저 자동화
mcp__playwright__navigate("https://example.com")
mcp__playwright__click("button#login")
```

**핵심 포인트**:
- ✅ Stateful (영구 상태 유지)
- ✅ JSON-RPC 2.0 프로토콜
- ✅ Bidirectional (서버→클라이언트 알림)
- ✅ Dynamic tool discovery

## 🔀 언제 무엇을 사용할까? (의사결정 트리)

```
사용자 요청 받음
       │
       ▼
   명령어인가? (/moai:*)
       │
       ├─ YES ─▶ COMMANDS 사용
       │
       ▼ NO
   상태 유지 필요?
       │
       ├─ YES ─▶ MCP SERVERS
       │
       ▼ NO
   read-only 탐색?
       │
       ├─ YES ─▶ AGENTS (Explore)
       │
       ▼ NO
   도메인 전문성 필요?
       │
       ├─ YES ─▶ AGENTS (expert-*)
       │
       ▼ NO
   지식만 필요?
       │
       ├─ YES ─▶ SKILLS
       │
       ▼ NO
   빠른 one-shot?
       │
       ├─ YES ─▶ SCRIPTS
       │
       ▼ NO
   복잡한 multi-step
       │
       └─▶ AGENTS (general-purpose)
```

## 🎓 실전 예제 3가지

### 예제 1: 새 기능 구현 (JWT 인증)

**시나리오**: 사용자 인증 기능 추가

**단계**:
```bash
# 1. SPEC 생성 (COMMANDS)
/moai:1-plan "User authentication with JWT"

# Alfred가 자동으로:
# - manager-spec 에이전트 호출 (AGENTS)
# - moai-foundation-spec 스킬 로드 (SKILLS)
# - SPEC-001 문서 생성
# - Git branch 생성 (git_strategy.mode에 따라)

# 2. TDD 구현 (COMMANDS)
/moai:2-run SPEC-001

# Alfred가 자동으로:
# - manager-tdd 에이전트 호출 (AGENTS)
# - expert-backend 에이전트 호출 (AGENTS)
# - moai-lang-python 스킬 로드 (SKILLS)
# - RED-GREEN-REFACTOR 사이클 실행
# - pre_commit 훅 자동 실행 (SCRIPTS)

# 3. 문서 동기화 (COMMANDS)
/moai:3-sync SPEC-001

# Alfred가 자동으로:
# - manager-docs 에이전트 호출 (AGENTS)
# - 문서 정리 및 PR 생성
```

**사용된 래핑 방법**:
- COMMANDS: /moai:1-plan, /moai:2-run, /moai:3-sync
- AGENTS: manager-spec, manager-tdd, expert-backend, manager-docs
- SKILLS: moai-foundation-spec, moai-lang-python
- SCRIPTS: pre_commit hook

### 예제 2: 코드베이스 탐색 (인증 코드 찾기)

**시나리오**: 기존 인증 로직 위치 파악

**단계**:
```python
# AGENTS (Explore) 사용 - Read-only, 빠른 실행
Task(subagent_type="Explore",
     prompt="Find all authentication handlers and middleware",
     thoroughness="medium")

# 결과: "Found in src/auth/handlers.py:42, src/middleware/auth.py:18"
```

**사용된 래핑 방법**:
- AGENTS: Explore (Haiku 모델, 빠른 read-only)

### 예제 3: 최신 React 19 API 문서 조회

**시나리오**: React 19 새 기능 학습

**단계**:
```typescript
// 1. MCP SERVERS 사용 - Stateful, 실시간 문서
mcp__context7__resolve-library-id("react")
// 결과: "/facebook/react"

// 2. 최신 버전 문서 가져오기
mcp__context7__get-library-docs("/facebook/react/v19.0.0")
// 결과: React 19 공식 API 문서 (최신)

// 3. 특정 주제 깊이 파고들기 (Stateful이므로 컨텍스트 유지)
mcp__context7__get-library-docs("/facebook/react/v19.0.0", topic="hooks")
// 결과: React 19 Hooks 상세 문서
```

**사용된 래핑 방법**:
- MCP SERVERS: context7 (Stateful, 실시간 API 문서)
