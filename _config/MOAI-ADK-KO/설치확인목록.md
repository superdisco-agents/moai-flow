# MOAI-ADK 설치 확인 목록

## Ghostty 터미널 폰트 설정 (필수 우선 단계)

MOAI-ADK를 설치하기 전에 Ghostty 터미널의 폰트를 올바르게 설정해야 합니다.

### Ghostty 설정 파일 위치
```bash
~/.config/ghostty/config
```

### 권장 폰트 설정
```bash
# Ghostty 설정 파일에 다음을 추가하세요
font-family = "SF Mono"
font-size = 14

# 또는 Nerd Font 사용 시
font-family = "JetBrainsMono Nerd Font"
font-size = 14

# 한글 지원을 위한 폴백 폰트
font-family-bold = "SF Mono Bold"
font-family-italic = "SF Mono Italic"
```

### 폰트 설정 확인
- [ ] **Ghostty 설정 파일이 존재하는지 확인**
  ```bash
  cat ~/.config/ghostty/config
  ```

- [ ] **폰트가 올바르게 렌더링되는지 확인**
  - Ghostty 터미널을 다시 시작합니다
  - 한글과 영문이 모두 잘 보이는지 확인합니다
  - 특수 문자와 아이콘이 제대로 표시되는지 확인합니다

## 빠른 참조 명령어

### 1. 최신 버전 확인
```bash
python3 _config/MOAI-ADK/scripts/check-latest-version.py

# 일반적인 플래그:
--update          # 새 버전이 있으면 자동 업데이트
--verbose         # 상세한 버전 정보 표시
--json            # JSON 형식으로 출력
--check-only      # 확인만 하고 업데이트 제안하지 않음
```

### 2. MCP 서버 확인
```bash
python3 _config/MOAI-ADK/scripts/verify-mcp-servers.py

# 일반적인 플래그:
--fix             # 설정 문제 해결 시도
--verbose         # 상세한 확인 단계 표시
--server <name>   # 특정 서버만 확인
--json            # JSON 형식으로 결과 출력
```

### 3. 설정 유효성 검사
```bash
python3 _config/MOAI-ADK/scripts/validate-config.py

# 일반적인 플래그:
--fix             # 일반적인 설정 문제 자동 수정
--strict          # 엄격한 유효성 검사 규칙 사용
--output <file>   # 유효성 검사 보고서를 파일로 저장
--verbose         # 상세한 유효성 검사 단계 표시
```

### 4. 빠른 상태 점검
```bash
python3 _config/MOAI-ADK/scripts/quick-doctor.py

# 일반적인 플래그:
--full            # 종합적인 상태 점검 실행
--fix             # 감지된 문제 해결 시도
--json            # JSON 형식으로 결과 출력
--quiet           # 오류와 경고만 표시
```

### 5. 사전 설치 확인
```bash
python3 _config/MOAI-ADK/scripts/pre-install-check.py

# 일반적인 플래그:
--clean           # 충돌을 자동으로 정리
--backup          # 정리하기 전에 기존 설정 백업
--dry-run         # 실제로 정리하지 않고 정리될 내용만 표시
--verbose         # 상세한 충돌 감지 표시
```

## 일반적인 설치 문제 및 해결 방법

### 문제: Python 버전이 너무 낮음
**해결 방법:**
```bash
# macOS (Homebrew 사용)
brew install python@3.11

# Ubuntu/Debian
sudo apt-get install python3.11

# 설치 확인
python3.11 --version
```

### 문제: 가상 환경이 활성화되지 않음
**해결 방법:**
```bash
# 올바른 활성화 스크립트를 사용하는지 확인
source .venv/bin/activate        # macOS/Linux (bash/zsh)
source .venv/bin/activate.fish   # Fish shell
.venv\Scripts\activate.bat       # Windows CMD
.venv\Scripts\Activate.ps1       # Windows PowerShell
```

### 문제: MCP 서버가 인식되지 않음
**해결 방법:**
```bash
# .mcp.json이 존재하고 유효한 JSON인지 확인
cat .mcp.json | python3 -m json.tool

# MCP 서버 확인 재실행
python3 _config/MOAI-ADK/scripts/verify-mcp-servers.py --fix
```

### 문제: 명령어가 Claude Code에 나타나지 않음
**해결 방법:**
```bash
# 명령어 파일이 존재하는지 확인
ls -la .claude/commands/moai/

# Claude Code CLI 재시작
# 그런 다음 테스트: /moai-status
```

### 문제: 권한 거부 오류
**해결 방법:**
```bash
# 스크립트를 실행 가능하게 만들기
chmod +x _config/MOAI-ADK/scripts/*.py

# 권한 확인
ls -la _config/MOAI-ADK/scripts/
```
